Команда dialog

Привет, мы – это dialog.

В блоге мы будем рассказывать о том какие функции и как использовать, что появилось нового, ну или просто делиться удачными кейсами, новыми интеграциями и возможностями. Начнем!

Что такое мессенджер dialog?

dialog – это удобный конструктор мессенджеров для iOS, Android, Linux, web и десктопов, а еще – это приложение для организации коммуникации в командах. Движок мессенджера dialog разрабатывался более двух лет и включает в себя помимо базового функционала современного мессенджера, звонки и совместимость с чатботами – платформа поддерживает создание ботов пользователями, всё как в Slack. Преимущество dialog – это то, что функции доступны юзерам бесплатно.

Тестировать публичную бету dialog вдоль и поперек нужно: в web – app.dlg.im и в приложениях (iOS, Android).

Нашими клиентами являются крупные компании и организации, которые устали от малопродуктивного общения с использованием нескольких каналов связи. Теперь все коммуникации таких клиентов происходят на внутренней безопасной и централизованной инфраструктуре dialog.

Присоединяйтесь!

Клиентская часть: dialog SDK Client и dialog SDK Server

Программистам стоит обратить внимание на SDK мессенджера dialog. Являясь классическим примером продукта, разработанного by developers for the developers, dialog SDK поможет быстро запустить собственный качественный мультиплатформенный месседжинговый сервис.

Платформа dialog масштабируется и имеет гибкие возможности кастомизации и интеграции, таким образом, может быть использована в качестве базы для создания любого специфического мессенджера по модели white label.

API мессенджера dialog проприетарен, а код написан с минимальным использованием open-source источников: никакого недокументированного поведения кода в dialog нет –  вы полностью контролируете всё, что происходит.

В зависимости от потребностей компании, dialog предлагает 2 хостинг-плана: cloud и self-hosted (on-premise или self-hosted instant messaging) решения, с оплатой за пользователя или за ограниченную лицензию, соответственно.

С помощью инструментов dialog Client SDK и dialog Server SDK, можно вносить любые изменения в интерфейс, логику и функционал приложения.

Поправить дизайн и интерфейс поможет Client SDK, а с помощью Server SDK происходит доработка функционала, интеграции с корпоративными внешними или внутренними сервисами и пр..

Готовый мессенджер на базе dialog: dialog-based App

Создание приложений на базе dialog проходит в три этапа:

  1. Выбор сервера и размещение платформы dialog.
  2. Создание/кастомизация приложения на базе dialog.
  3. Размещение dialog-based App в сторы.

Выбирая dialog в качестве основы для создания собственного мессенджера, вы получаете готовый, удобный и быстрый продукт, который уже включает в себя основные функции и по желанию может быть доработан как силами команды dialog, так на вашей стороне:

  • Чаты
  • Групповые чаты
  • Каналы
  • Звонки
  • Чатботы
  • Приглашения
  • Регистрация по e-mail / номеру телефона
  • Профили групп / пользователей
  • Голосовые сообщения
  • Отправка файлов
  • Обмен местоположением
  • Административная панель
